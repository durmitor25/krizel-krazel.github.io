⚡ Kako radi — korak po korak

Kada pokrneš python main.py, dešava se sljedeće:

1

Čitanje konfiguracije iz baze

configdb.py se spoji na PostgreSQL bazu i pročita dvije tabele: scraping (lista svih izvora — SURS, ZPIZ, GURS, PK) i output_path (putanje foldera na disku gdje se snimaju fajlovi). Automatski spoji ove dvije tabele — svaki izvor dobije stvarnu putanju foldera.

💡 Ako dodaš novi izvor u tabelu scraping, sistem ga automatski vidi pri sljedećem pokretanju — bez ikakvog mijenjanja koda.
2

Prolaz kroz sve aktivne izvore

main.py prođe kroz svaki red iz tabele scraping. Ako je red označen kao enabled = false, preskoči ga. Za svaki aktivan red pozove registry koji odlučuje kako da preuzme fajl.

💡 Možeš privremeno isključiti bilo koji izvor u bazi postavljanjem enabled = false — bez brisanja reda.
3

Registry bira pravi handler

handlers/__init__.py gleda kolonu handler u tom redu i bira odgovarajuću funkciju. Na primjer ako piše direct — pozove direct.py, ako piše api_then_dl — pozove api_then_dl.py. Ovo se zove registry pattern — kao telefonski imenik, tražiš po imenu i dobiješ broj.

💡 Ako handler nije prepoznat (npr. neko unese pogrešan naziv) — sistem logovira SKIP i nastavlja dalje. Ne pada, ne staje.
4

Handler preuzima fajl

Svaki handler radi istu stvar ali na drugačiji način — gradi URL, šalje HTTP request, prima sadržaj i snima ga na disk. Veličina fajla se mjeri odmah. Rezultat (OK ili ERROR) se vraća nazad u main.py.

💡 Greška na jednom fajlu ne prekida ostale. Sistem nastavi sa sljedećim izvorom i na kraju prikaže rezime.
5

Fajl se snima u LANDING folder

Preuzeti fajl se snima u odgovarajući LANDING subfolder. Naziv fajla se uvijek gradi po istoj konvenciji: MMYYYY_IZVOR_ID.ekstenzija. Na primjer: 032026_SURS_05A1002S.px. Folder se kreira automatski ako ne postoji.

6

Summary rezime na kraju

Nakon što svi izvori budu obrađeni, main.py ispiše rezime u log fajl i terminal. Vidiš svaki fajl — da li je OK ili ERROR — i kolika mu je veličina. Na dnu je ukupan broj uspješnih i neuspješnih preuzimanja.

7

Unzip → Convert → Check (opciono)

Ovi koraci se pokreću odvojeno (ručno ili kroz Airflow). unzipper.py raspakuje ZIP fajlove iz LANDING u BRONZE. converters/ konvertuje .px i .json u Parquet format. check.py pregleda log i javlja ako ima grešaka.

🔧 Handleri — tipovi preuzimanja

Svaki handler prima isti row dict iz baze i vraća rezultat. Jedina razlika je kako se gradi URL i kako se preuzima fajl.

Handler URL logika Koristi se za Posebnosti
direct url + file_id SURS (.px fajlovi) Najjednostavniji — direktan GET request
direct_ext url + file_id + "." + type_file ZPIZ (.xlsx), PK (.zip) Isti kao direct, samo dodaje ekstenziju na kraj URL-a
direct_full url (kompletan, ništa se ne dodaje) GURS (.json fajlovi) file_id se ne dodaje na URL — koristi se samo za naziv fajla na disku. JSON se snima kao tekst (UTF-8)
api_then_dl 2 koraka: META API poziv → dobij pravi URL → preuzmi GURS (.zip arhive) Korak 1: GET na API sa api_params iz baze → response JSON sadrži "url" polje. Korak 2: streaming download ZIP fajla sa tog URL-a
💡 Kada dođe novi tip izvora koji ne odgovara ni jednom handleru: 1. Kreiraj handlers/novi_handler.py sa download(row) -> dict funkcijom
2. U handlers/__init__.py dodaj import i jedan red u REGISTRY dict
3. main.py se ne mijenja!

✅ Šta znači svaki status: OK — fajl uspješno preuzet, vidljiva je i veličina u MB

➕ Kako dodati novi izvor

Sistem je dizajniran tako da dodavanje novog izvora ne zahtijeva mijenjanje koda — u većini slučajeva. Ispod su opisani svi mogući scenariji — šta može odmah, a šta treba developera.

✅ Može bez ikakvog koda — samo uneseš red u bazu

1

Direktan link na fajl (xlsx, csv, pdf, px...)

Imaš URL koji kad otvoriš u browseru — fajl se odmah skida. Nema logina, nema klikanja. Samo klikneš link i fajl se preuzme.

Primjer: https://sko.gov.si/data/izvjestaj2025.xlsx

Koristi direct_ext ako URL ne sadrži ekstenziju:
  url = https://sko.gov.si/data/ + file_id = izvjestaj2025 + type_file = xlsx

Koristi direct_full ako je URL već kompletan sa ekstenzijom:
  url = https://sko.gov.si/data/izvjestaj2025.xlsx (file_id samo za naziv fajla)
2

Direktan link ali server nema validan SSL certifikat

Isti kao gore, ali browser ti možda prikaže upozorenje "Your connection is not private". Fajl se može skinuti, samo treba reći Pythonu da ignoriše SSL provjeru.

Primjer: GURS json i PK fajlovi imaju ovaj problem.

Postavi verify_ssl = false u tabeli za taj red. Sve ostalo ostaje isto.
3

API koji vrati JSON sa download linkom unutra

Zoveš jedan URL (API), on ti vrati JSON odgovor koji sadrži drugi URL — pravi link za download. Tek taj drugi link skineš. Može imati filtere kao parametre (godina, država...).

Primjer: GURS zip fajlovi rade ovako.

Koristi api_then_dl handler.
Filtere upiši u kolonu api_params kao JSON:
{"filterParam": "DRZAVA", "filterValue": 1, "filterYear": 2025}
Ako nema filtera — ostavi api_params prazno (NULL).
4

Isti tip fajla kao postojeći izvor ali drugi URL

Npr. SURS ima novi dataset koji još nije u sistemu, ali radi identično kao ostali SURS fajlovi. Samo kopiraš pattern i promijeniš URL i file_id.

Pogledaj postojeće redove u bazi za isti source_name — koristi iste postavke, samo promijeni url, file_id i description.

⚠️ Treba developera — novi handler u kodu

Sljedeći scenariji ne mogu raditi sa postojećim 4 handlera. Treba napisati novi handler fajl — što je posao developera, ali ne traje dugo (30-60 minuta).

!

Login / autentikacija prije downloada

Stranica zahtijeva da se prijaviš (username + password) prije nego možeš skinuti fajl. Browser pamti session, ali Python to ne radi automatski.

Primjer: Portal koji traži prijavu, interni sistemi, privatni portali.

Treba novi handler koji koristi requests.Session() — prvo se prijavi, pa preuzme fajl. Developer napiše handlers/session_dl.py.
!

OAuth / API token autentikacija

API zahtijeva Bearer token ili API ključ u headeru. Tipično za moderne API-je (GitHub, Google, razne državne platforme nove generacije).

Primjer: Authorization: Bearer eyJhbGc... u HTTP headeru.

Treba novi handler koji dodaje Authorization header u request. Token se može čuvati u bazi ili u environment varijabli.
!

Stranica gdje treba kliknuti dugme da se pokrene download

Fajl se ne skida direktno sa URL-a — treba otvoriti web stranicu, možda odabrati opcije iz dropdowna ili checkboxova, i tek onda kliknuti "Download". Ovo je tzv. web scraping sa interakcijom.

Primjer: Portal gdje birate godinu iz padajućeg menija pa kliknete Export.

Treba Selenium ili Playwright biblioteka koja kontroliše browser. Ovo je složeniji handler — developer treba 2-4 sata.
!

FTP server

Fajlovi su na FTP serveru (adresa počinje sa ftp://). Protokol je potpuno drugačiji od HTTP/HTTPS koji trenutni handleri koriste.

Treba novi handler koji koristi ftplib Python biblioteku. Relativno jednostavno za developera — oko 30 minuta.
!

URL koji sadrži dinamički datum

URL se mijenja svaki dan/mjesec/godinu automatski. Na primjer svaki mjesec novi fajl ima datum u imenu i nema fiksni file_id.

Primjer: https://portal.gov.si/data/2026/03/report.xlsx — sljedeći mjesec biće /2026/04/

Treba novi handler koji gradi URL dinamički na osnovu trenutnog datuma. Može se riješiti i sa api_params ako API prihvata datum kao parametar.
🔧 Kako developer dodaje novi handler (kratki vodič) Kad dođe scenarij koji ne mogu pokriti postojeći handleri, developer uradi samo ovo:

1. Kreira novi fajl handlers/naziv_handlera.py sa download(row) → dict funkcijom
2. U handlers/__init__.py doda dvije linije — import i red u REGISTRY dict
3. main.py, configdb.py i svi ostali fajlovi se NE mijenjaju

Nakon toga korisnik u bazi samo upiše novi naziv handlera u kolonu handler i sistem odmah radi.
